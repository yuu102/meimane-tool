# 🍁 めいまねつーる

Version 1.3.1

メイプルストーリーMのキャラクター管理ツールです。ES Modules対応の静的サイトとして、GitHub PagesまたはローカルHTTPサーバーで利用します。

## スクリーンショット

_スクリーンショットは後から追加してください。_

## 起動方法

ES Modulesを使用しているため、`file://` で `index.html` を直接開くと、ブラウザのセキュリティ制限により動かない場合があります。GitHub Pagesへ公開して利用するか、ローカルHTTPサーバーから開いてください。

例: プロジェクトフォルダで `python -m http.server 8000` を実行し、`http://localhost:8000/` をブラウザで開きます。

## 使い方

1. 右下の `＋` でキャラクターを追加します。レベル入力範囲は **1〜300** です。職業を選ぶと系列は自動設定されます。
2. 上部サマリーの「キャラ」「完了」「残り」を押すと、全キャラ・完了キャラ・未完了キャラへ表示を切り替えられます。検索は選択中の表示に対して適用されます。
3. 前日終了時レベル／EXPと日課終了後EXPを0〜100%・小数第2位までで記録できます。前日終了時レベルが空なら現在レベルを使用します。
4. カードを選ぶと詳細画面が開き、EXP履歴とデイリーのチェックを確認できます。すべてチェック済みになると自動で「完了」になります。
5. 並び替え、完了キャラの表示切替、JSONバックアップ／復元は設定画面から操作します。

## キャラクターの手動並び替え

設定画面の「キャラクター並び替え」から、↑↓ボタンでキャラクターの順番を変更できます。保存時に `order` を0から振り直し、登録順はこの手動順として表示されます。キャンセル時は変更されません。

- 新規キャラクターは末尾へ追加されます。
- 削除後は残ったキャラクターの `order` を連番へ自動調整します。
- 旧データや旧バックアップに `order` がない場合は、読み込んだ配列の順番を維持して補完します。

カードを約600ms長押ししても、同じ手動並び替えモードへ入れます。カード上の↑↓で一時的に順番を変更し、**保存**を押したときだけ `order` へ反映します。**キャンセル**は変更を破棄します。

長押し並び替えは「全キャラ表示」「登録順」「検索語なし」のときだけ利用できます。完了／残り表示中、検索中、または他の並び替えを選んでいるときは開始しません。設定画面の「キャラクター並び替え」も引き続き利用でき、同じ `order` を操作します。

## レベルアップが近い順

設定の並び替えで「レベルアップが近い順」を選ぶと、残りEXPが少ないキャラクターから表示します。日課終了後EXP（`afterDailyExp`）が入力されている場合はそれを優先し、未入力なら前日終了EXP（`previousExp`）を使用します。

```text
残りEXP = 100 - 使用するEXP
```

残りEXPが同じ場合は、手動 `order` 順を維持します。

## 今日の進捗

上部の「今日の進捗」は、検索・表示モード・完了キャラ非表示に関係なく、全キャラクターの全デイリーを対象に集計します。チェック済み数／全デイリー数と整数の達成率を表示し、デイリーのチェックやテンプレート変更直後に更新されます。全デイリーがない場合は `0 / 0`、`0%` と表示します。

## iPhone向けカード表示

カードは長いキャラクター名を見やすくするため、名前を1行目、`Lv.` と職業バッジを2行目に表示します。職業バッジは系列ごとに戦士＝赤、魔法使い＝青、弓使い＝緑、盗賊＝紫、海賊＝黄、ハイブリッド＝グレーで表示されます。

EXPはカード上で `前日 45.67%`、日課終了後を記録済みの場合は `日課後 52.81%` のように短く表示します。

## iPhone Safariでの操作

- カード長押しはTouch Eventsを優先し、約600msで並び替えモードへ入ります。カードのテキスト選択と長押しメニューは抑止し、縦スクロールで12px以上動かした場合は長押しをキャンセルします。
- ダイアログ内の入力欄は16px以上にし、iPhone Safariの入力時自動拡大を防ぎます。ユーザーのピンチズームは制限しません。
- ダイアログ表示中は背景のスクロール位置を固定し、ダイアログ内部だけを慣性スクロールできます。詳細から編集へ切り替える場合も、背景位置を維持します。

## 日課終了後EXPと獲得EXP

`afterDailyExp` を入力すると、カードと詳細画面にその日の獲得EXP相当を表示します。

```text
（現在level − previousLevel）× 100 + afterDailyExp − previousExp
```

例: 開始 `Lv.200 / 95.00%`、終了 `Lv.201 / 10.00%` の場合は `+15.00%` です。現在レベルが開始レベルより低い場合など、計算できないデータでは獲得量を表示しません。

デイリー自動リセット時、`afterDailyExp` が入力されていれば、それを次回の `previousExp` へ、現在レベルを `previousLevel` へ引き継いだ後に日課終了後EXPを空にします。未入力の場合は開始値を変更しません。

## デイリーテンプレート管理

設定画面の「デイリー編集」では、以下を一時編集できます。

- デイリーの追加・削除・名称変更・上下並び替え
- デイリー名は最大40文字。空名は保存できません。同名のデイリーも利用できます。
- **保存**を押したときだけ、変更内容が全キャラクターへ反映されます。キャンセル時は変更されません。
- 同一テンプレートの名称を変更しても、テンプレートIDで対応付けるためチェック状態を維持します。
- 追加した項目は全キャラクターへ未チェックで追加され、削除した項目は全キャラクターから削除されます。
- テンプレートに属さない過去データ由来の独自デイリーは、テンプレートの後ろに保持されます。

## 設定とLocalStorage

キャラクターは `meimane.characters`、設定は `meimane.settings` に保存されます。

```json
{
  "sortMode": "favorite",
  "autoDailyReset": true,
  "lastResetDate": "2026-07-15",
  "hideCompleted": false,
  "dailyTemplate": [
    { "id": "template-uuid", "title": "デイリークエスト" },
    { "id": "template-uuid-2", "title": "遠征" }
  ]
}
```

- `sortMode`: `default`、`favorite`、`level`、`name` のいずれか。
- `hideCompleted`: `true` の場合、完了キャラを一覧から非表示にします。サマリーは全キャラを対象にします。
- `autoDailyReset`: `true` なら日付が変わって最初にアプリを開いたとき、すべてのデイリーを未チェックへ戻します。OFFからONへ変更した当日はリセットせず、翌日から開始します。
- `dailyTemplate`: ID付きのテンプレート配列です。IDは名称変更・並び替え・反映時の内部識別に使われます。

## JSONバックアップ／復元

バックアップはVersion 1.3.1形式で、キャラクターと設定をまとめて保存します。復元では次の旧形式も利用できます。

- キャラクター配列だけのJSON
- `{ "characters": [] }` 形式
- `settings` を持たないJSON
- `dailyTemplate` が文字列配列の設定
- `templateId` を持たない `dailies[]`

新形式のバックアップに設定が含まれる場合は、キャラクターと設定の両方を復元します。復元時にテンプレートとデイリーを正規化し、完了状態も再計算します。

## 既存データ互換性

- 旧文字列配列の `dailyTemplate` は、読み込み時にID付き形式へ移行します。
- 既存デイリーに `templateId` がなければ、同名テンプレートへ並び順を考慮して対応付けます。一致しないデイリーは保持します。
- 旧データ由来の `デイリー 1`、`デイリー 2`、`デイリー 3` だけで構成されたデイリーは、現在のテンプレートへ置き換え、チェック状態を順番に引き継ぎます。
- `デイリー1〜3` と `デイリー 1〜3` は、独自デイリーが混在していても仮デイリーだけを現在テンプレートへ移行します。余った仮デイリーは削除し、独自デイリーは残します。
- IDなしデータ、旧 `exp`、旧 `daily.progress / goal`、旧職業名を読み込めます。
- `previousExp` は0〜100%へ丸め、小数第2位までに正規化します。
- 旧データに `previousLevel` がない場合は現在の `level` を使用し、`afterDailyExp` がない場合は空文字として扱います。
- 現行キャラクターのEXP項目は `level`、`previousLevel`、`previousExp`、`afterDailyExp` です。
- 現行キャラクターには手動並び順の `order` も保存されます。

```text
meimane-tool/
├── index.html       # type="module" で app.js を読み込む画面

├── dialogs.js        # キャラ／詳細／設定ダイアログ
├── backup.js         # JSONバックアップ・復元
├── utils.js          # UUID・入力正規化・ローカル日付
├── README.md
└── CHANGELOG.md
```
