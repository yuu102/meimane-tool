# めいまねつーる

ブラウザだけで使えるキャラクター管理ツールです。ビルドやサーバーは不要です。`index.html` をブラウザで開くだけで利用できます。

## Phase 3.2 の変更

- EXP表示を `EXP 12.34%` の形式へ変更しました。これは前日終了時のEXPを、次レベルに必要なEXPに対する割合で表したものです。次のレベルまでの残り割合も小数第2位まで表示します。
- 職業は自由入力ではなく、職業一覧から選択する方式へ変更しました。
- 系列は選択した職業から自動設定され、職業バッジの色に反映されます。
- ゼノンは盗賊／海賊のハイブリッド職として扱い、バッジを2色のグラデーションで表示します。

既存の一覧にある職業名が職業一覧に存在しない場合も、編集時に「既存職業」として選択肢へ残るため、データは失われません。

## Phase 3.1 の変更

- カードは概要表示だけに変更しました。カードを選ぶと詳細画面が開き、デイリーのチェックは詳細画面内で行います。
- 職業に加えて、系列（戦士／魔法使い／盗賊／弓使い／海賊）を追加しました。系列に応じて職業バッジの色が変わります。
- EXPは `現在EXP / 必要EXP` を残したまま、進捗率を小数第2位まで表示します。
- デイリーが1件以上あり、すべてにチェックが入るとキャラクターは自動で「完了」になります。どれかを外すと「未完了」に戻ります。

### 詳細画面

カードを押すと詳細画面が開きます。ここからデイリーのチェック、キャラクター編集、デイリー編集を利用できます。デイリーのチェックボックスはカードには表示されません。

## Phase 2 の追加機能

- **EXPバー**: 登録済みのEXPを現在レベルの次レベル必要EXPに対する進捗として表示します。必要EXPは `100 + レベル × 25` で算出しており、将来のレベルアップ処理で `getExpGoal()` を差し替えられます。
- **デイリー進捗**: カードごとに `0 / 3` の進捗を表示し、`−` と `＋` で更新します。進捗は0〜目標値の範囲に制限され、LocalStorage・JSONバックアップに保存されます。
- **状態別の見た目**: 完了カードは緑系、お気に入りカードは金色の強調表示になります。
- **ゲーム風UI**: カードのホバー浮遊アニメーション、EXPバー、次レベルまでのEXP表示を追加しました。

Phase 2では `style.css` をゲーム風UIのために更新しています。`index.html` は変更していません。

## Phase 3 の追加機能

- EXPバー右側に進捗率（`%`）を表示し、`現在EXP / 必要EXP` 表示は維持しました。
- キャラクターに任意の `job`（職業）を追加しました。追加・編集ダイアログで入力でき、カードでは名前の横に表示され、検索対象にも含まれます。
- デイリーを単一の進捗値から、個別クエストの `dailies[]` へ移行しました。各クエストはチェックボックスで完了を切り替えられます。
- カードの `デイリー編集` から専用画面を開き、クエストの追加・削除・上下並び替えができます。

## デイリーの使い方

1. カード内のチェックボックスで、その日のクエストを完了／未完了にします。
2. `デイリー編集` を選ぶと、キャラクター編集とは別のクエスト管理画面が開きます。
3. クエスト名を入力して追加し、`↑` / `↓` で並び替え、`削除` で取り除きます。

## Phase 1.1 の機能

- キャラクターの追加・編集・削除
- キャラクター名のリアルタイム検索
- UUIDベースの識別子による安全な編集・削除（検索中も同じ）
- LocalStorageへの自動保存と起動時の自動復元
- 完了 / 未完了の切替と、合計・完了・残り件数の自動集計
- お気に入りの切替（カード上部の `★` / `☆`）
- レベルの全角数字から半角数字への自動変換と数値検証（1〜999）
- 追加・編集用のダイアログ

## Phase 1.2 の追加機能

- 検索結果を「お気に入り順」「Lv順（降順）」「名前順（昇順）」に表示できます。並び替えは表示だけに適用され、保存済みの登録順は変更しません。
- `JSONバックアップ` ボタンで、すべてのキャラクターデータを1つのJSONファイルに保存できます。
- `JSON復元` ボタンで、Phase 1.2のバックアップ、または旧形式のキャラクター配列JSONを復元できます。復元時は現在のデータを置き換える確認が表示されます。

並び替えとバックアップ操作のUIは、`index.html` を変更せず `app.js` が検索欄の下に追加します。

## 使い方

1. 右下の `＋` でキャラクターを追加します。
2. カード本体を選択すると編集ダイアログが開きます。
3. カード上部の `★` / `☆` でお気に入りを切り替えます。
4. カード上部の `○ 未完了` / `✓ 完了` で完了状態を切り替えます。
5. 上部の検索欄に入力すると、名前に一致するカードだけが表示されます。検索結果からも編集・削除・状態切替ができます。

## LocalStorageのデータ設計

保存キーは `meimane.characters` です。ブラウザ内に次のようなJSON配列で保存されます。

```json
[
  {
    "id": "b0a5d4c2-...",
    "name": "キャラクター名",
    "level": "80",
    "previousExp": "123.45",
    "job": "ヒーロー",
    "series": "戦士",
    "dailies": [
      { "id": "daily-uuid-1", "title": "採取", "checked": true },
      { "id": "daily-uuid-2", "title": "討伐", "checked": false }
    ],
    "completed": false,
    "favorite": false
  }
]
```

- `id`: UUID。表示順や検索結果の番号には依存しません。
- `completed`: 完了状態。`true` のレコードが「完了」に集計されます。
- `favorite`: お気に入り状態。`true` ならカード上部に `★` が表示されます。
- `job`: 任意の職業名です。
- `series`: 職業から自動設定される系列です。`戦士`、`魔法使い`、`盗賊`、`弓使い`、`海賊`、`ハイブリッド`、または空文字列です。
- `dailies`: デイリークエストの配列です。各要素は一意ID、表示名、チェック状態を持ちます。

Phase 3では旧 `daily.progress` / `daily.goal` を `dailies[]` に移行します。たとえば `progress: 2, goal: 3` は、3件のクエストのうち先頭2件がチェック済みの状態になります。旧データは読み込み・JSON復元時に自動移行され、LocalStorageへPhase 3形式で保存し直されます。

バックアップJSONは、メタデータとキャラクター一覧を含む次の形式です。

```json
{
  "app": "meimane-tool",
  "version": "3.2",
  "exportedAt": "2026-07-15T00:00:00.000Z",
  "characters": []
}
```

## 既存データとの互換性

Phase 1以前のデータもそのまま読み込めます。IDがない旧データには読み込み時にUUIDを補完します。`completed` と `favorite` がない既存レコードは、ともに `false` として扱われます。旧フィールド `exp` も `previousExp` として移行します。未知のフィールドは保持されるため、将来の機能追加でデータを失いません。

## アーキテクチャ

`app.js` は次の責務に分けています。

- 正規化・UUID生成: 入力値と旧データを安全な形式にそろえます。
- Storage: 読み込み、旧キーの移行、保存を担当します。
- Render: `renderCard()`、`renderSummary()`、`renderEmpty()` に分け、描画を明確化しています。
- State actions: お気に入り・完了の切替、保存、削除をIDで処理します。
- Dialog/events: フォーム、ダイアログ、キーボード入力を管理します。
- Backup: `exportBackup()` と `importBackup()` が、データのダウンロードと検証済み復元を担当します。

すべての状態変更は `saveCharacters()` の後に `render()` を呼びます。そのため、検索中でも対象カードのUUIDで正しく更新されます。

## 注意

データは使用中のブラウザ・端末にだけ保存されます。ブラウザのサイトデータを削除すると失われるため、重要なデータは別途控えてください。
