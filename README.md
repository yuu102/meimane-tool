# めいまねつーる — Phase 3.2

ブラウザだけで使えるキャラクター／クエスト管理ツールです。ビルドやサーバーは不要で、`index.html` をブラウザで開くだけで使えます。

## 主な機能

- キャラクターの追加・編集・削除、リアルタイム検索
- UUIDによる安全な編集・削除
- お気に入り、完了状態、名前・Lv・お気に入り順ソート
- LocalStorageへの自動保存、JSONバックアップ／復元
- 個別デイリークエストの追加・削除・並び替え・チェック
- 全デイリー達成時の自動完了判定
- 職業の選択と、系列に応じたカラー表示

## 使い方

1. 右下の `＋` からキャラクターを追加します。
2. 職業は一覧から選ぶと、系列が自動設定されます。ゼノンは盗賊／海賊のハイブリッド色で表示されます。
3. 前日終了EXPには `0`〜`100` の割合を入力します。たとえば `45.67` はカード上で `EXP 45.67%` と表示されます。
4. カードを選ぶと詳細画面が開きます。詳細画面でデイリーをチェックし、キャラ編集・デイリー編集へ進めます。
5. デイリーが1件以上あり、すべてにチェックが入ると自動的に「完了」になります。どれかを外すと「未完了」に戻ります。
6. 上部の検索欄、並び替え、JSONバックアップ／復元を必要に応じて使用します。

カードにはデイリーのチェックボックスを表示しません。チェック操作は詳細画面で行います。

## EXPの扱い

`previousExp` は実数EXPではなく、**前日終了時のEXP割合**です。

- 保存範囲: `0`〜`100`
- 小数: 最大2桁に正規化
- 表示例: 入力 `45.67` → `EXP 45.67%`
- EXPバー幅: `45.67%`
- 次のレベルまで: `54.33%`

旧データやバックアップの `previousExp` / `exp` も、読み込み時にこの範囲へ丸めて維持します。

## 職業と系列

職業は戦士、魔法使い、弓使い、盗賊、海賊の一覧から選択します。系列は職業から自動決定されるため、手動変更は不要です。ゼノンのみハイブリッド系列として扱います。

既存データに一覧外の職業名がある場合も、編集ダイアログに「既存職業」として残るため、データは失われません。

## データ構造

データはLocalStorageキー `meimane.characters` に保存されます。

```json
[
  {
    "id": "character-uuid",
    "name": "キャラクター名",
    "job": "ヒーロー",
    "series": "戦士",
    "level": "80",
    "previousExp": "45.67",
    "dailies": [
      { "id": "daily-uuid", "title": "採取", "checked": true }
    ],
    "completed": false,
    "favorite": false
  }
]
```

旧 `daily.progress / goal` 形式は `dailies[]` へ自動移行されます。旧版のIDなしデータ、`exp` フィールド、職業・系列・デイリーを含むバックアップにも互換対応しています。
