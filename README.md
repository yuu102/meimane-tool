# めいまねつーる — Phase 1

ブラウザだけで使えるキャラクター管理ツールです。ビルドやサーバーは不要です。

## 使い方

1. `index.html` をブラウザで開きます。
2. 右下の `+` ボタンからキャラクター名・レベル・前日EXPを入力して保存します。
3. カードを選択すると編集できます。編集ダイアログ内の削除ボタンで削除できます。
4. 上部の検索欄は入力と同時にキャラクター名を絞り込みます。絞り込み中のカードも通常どおり編集・削除できます。

## Phase 1 の機能

- キャラクターの追加、編集、削除
- キャラクター名のリアルタイム検索
- UUIDベースの識別子による管理
- LocalStorageへの自動保存と起動時の自動復元
- レベル入力の全角数字から半角数字への自動変換
- レベルの数値検証（1〜999）
- 合計・完了・残りの件数表示
- `dialog` を用いた追加・編集画面

`index.html` と `style.css` は変更せず、機能は `app.js` に実装しています。

## データ設計

データはブラウザの LocalStorage キー `meimane.characters` に、JSON配列として保存します。

```json
[
  {
    "id": "b0a5d4c2-...",
    "name": "キャラクター名",
    "level": "80",
    "previousExp": "123.45",
    "daily": {},
    "completed": false
  }
]
```

- `id`: UUID。表示順や検索結果の配列番号ではなく、編集・削除の対象をこの値で特定します。
- `name`: キャラクター名。
- `level`: `1`〜`999` の文字列。
- `previousExp`: 任意の前日EXP。
- `daily`: Phase 2 で日別データを追加するための予約領域です。
- `completed`: 完了数の集計用フラグです。Phase 1の画面には切替UIがないため、新規作成時は `false` です。

旧版の保存データにIDが無い場合も、初回読み込み時にIDを補完して安全に移行します。読み込み不能なJSONは無視し、空の一覧として起動します。

## 実装方針

`app.js` は、データの読み書き、入力の正規化、描画、ダイアログ制御、イベント登録を分けています。すべての画面更新は `render()` を経由するため、検索状態でもIDで正しいキャラクターを編集・削除できます。

カード内の `0 / 0` は既存画面のデイリー表示枠を保つための値です。Phase 2で `daily` を使う実装を追加できます。

## 注意

LocalStorageのデータはブラウザ・端末ごとに保存されます。ブラウザのサイトデータを削除すると一覧も消えるため、重要なデータは将来のバックアップ機能追加まで別途控えてください。
