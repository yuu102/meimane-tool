# Changelog

## Version 1.0.0

- Phase 1〜Phase 3.2を統合した正式版リリース。
- キャラクター管理、職業・系列管理、EXP割合管理、デイリー管理を提供。
- JSONバックアップ／復元、設定保存、デイリー自動リセット、ES Modulesに対応。
- レベル上限を300へ変更。
- デイリー自動リセットをOFFからONへ変更した当日は、リセットせず翌日から開始するよう改善。

## Phase 3.2 — モジュール化・設定・日次リセット

- 正規化時の完了状態を保存値ではなくデイリー達成状況から再計算。
- カードに詳細画面用のARIA属性、並び替え操作に表示ラベルとARIAラベルを追加。
- JSON復元時に各キャラクター要素のオブジェクト形式と`name`文字列を検証。
- ES Modules利用時のHTTPサーバー／GitHub Pagesでの起動方法をREADMEへ追記。

- `app.js` を責務別のES Modulesへ分割し、`index.html` を `type="module"` に変更。
- `meimane.settings` を追加。並び替え方法、デイリー自動リセット、最終リセット日を保存。
- 設定ダイアログと「設定」ボタンを追加。
- 日付が変わった最初の起動時にデイリーを自動リセットする機能を追加。
- 未使用の旧daily-progress系CSSと旧単一ファイル実装を整理。

## Phase 3.2

- `previousExp` を実数EXPではなく、前日終了時のEXP割合（0〜100%、小数第2位）として正規化。
- EXPバー、`aria-valuemax`、次のレベルまでの表示をすべて割合基準へ変更。
- 必要EXPの固定計算と未使用の `toggleCompleted()` を削除。
- READMEの操作説明を詳細画面方式へ更新し、HTMLタイトルを「めいまねつーる」へ変更。

- EXPを「前日終了時のEXP％」として表示し、次レベルまでの残り割合を小数第2位まで表示。
- 職業を選択式に変更し、戦士・魔法使い・弓使い・盗賊・海賊の職業一覧を追加。
- 選択職業から `series` を自動設定し、職業バッジを系列別に色分け。
- ゼノンをハイブリッド職として扱い、盗賊／海賊の2色バッジを追加。
- 既存の自由入力職業は編集時も保持する互換処理を追加。

## Phase 3.1

- カード内のデイリーチェック表示を廃止し、カード選択で開く詳細画面へ移動。
- 詳細画面からデイリーのチェック、キャラクター編集、デイリー編集を利用可能に変更。
- 職業に `series`（戦士／魔法使い／盗賊／弓使い／海賊）を追加し、職業バッジを系列別に色分け。
- EXP進捗率を小数第2位まで表示。`EXP 現在値 / 必要値` は維持。
- デイリーがすべてチェック済みなら自動で完了、1つでも未チェックなら未完了へ自動更新。

## Phase 3

- EXPバーに進捗率（%）を追加し、現在EXP / 必要EXP表示を維持。
- `character.job` を追加。追加・編集、カード表示、検索に対応。
- `daily` の単一進捗形式を `dailies[]` のクエスト配列形式へ移行。
- カード上でクエストをチェックできるデイリー表示を追加。
- 専用の「デイリー編集」ダイアログを追加。クエストの追加・削除・上下並び替えに対応。
- 旧 `daily.progress / goal` のLocalStorageデータとJSON復元データを、個別クエストへ自動移行。

## Phase 2

- EXPバー、現在EXP、次レベルまでの必要EXPをカードに追加。
- デイリー進捗（初期値 `0 / 3`）と、カード上の増減ボタンを追加。
- 完了カードの緑色表示、お気に入りカードの金色強調表示を追加。
- カードのホバー浮遊アニメーション、フォーカス表示、ゲーム風のカード・EXPバー表現を追加。
- 既存のデイリーデータを `daily.progress` / `daily.goal` 形式へ正規化し、既存データ・JSON復元との互換性を維持。

## Phase 1.2

- お気に入り順、Lv順（降順）、名前順（昇順）の表示ソートを追加。
- JSONバックアップのダウンロード機能を追加。
- JSON復元機能を追加。Phase 1.2形式と旧形式の配列JSONに対応し、重複UUIDは復元時に補正。
- 検索欄の下に、HTML/CSSを変更せず操作バーを動的に追加。

## Phase 1.1

- `completed` 状態の切替を追加し、合計・完了・残りの件数を正しく集計するように変更。
- `favorite: boolean` を追加。カード上部の `★` / `☆` で切り替え、LocalStorageへ保存。
- 描画処理を `renderCard()`、`renderSummary()`、`renderEmpty()` に分割。
- 空状態やダイアログなどの表示文言を定数化。
- LocalStorageの読み込みと `normalizeCharacter()` を整理し、旧データの `id`、`completed`、`favorite`、`exp` を安全に移行。
- 保守しやすいコメントを追加し、UUIDを基準とする検索中の編集・削除を維持。
- READMEをPhase 1.1の機能、データ構造、利用方法、アーキテクチャに更新。
